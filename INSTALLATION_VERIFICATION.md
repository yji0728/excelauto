# 설치 확인 체크리스트 (Installation Verification Checklist)

Excel 자동화 앱이 제대로 설치되고 작동하는지 확인하기 위한 체크리스트입니다.

## 📋 설치 전 체크리스트

### 1. 파일 구조 확인

다음 파일들이 모두 존재하는지 확인하세요:

```
excelauto/
├── index.html                    ✓ 메인 애플리케이션
├── app.js                        ✓ JavaScript 로직
├── README.md                     ✓ 전체 가이드
├── QUICKSTART.md                 ✓ 빠른 시작
├── .gitignore                    ✓ Git 설정
├── lib/
│   ├── xlsx.full.min.js         ⚠️ 라이브러리 (설치 필요)
│   └── README.md                ✓ 설치 가이드
└── templates/
    ├── README.md                ✓ 템플릿 가이드
    └── SAMPLE_DATA.md           ✓ 샘플 데이터 가이드
```

### 2. SheetJS 라이브러리 확인

**중요:** 이 단계는 필수입니다!

- [ ] `lib/xlsx.full.min.js` 파일이 존재하는가?
- [ ] 파일 크기가 **800KB ~ 1.2MB** 인가?
  - 1KB 미만이면 stub 파일이므로 실제 라이브러리로 교체 필요
- [ ] `lib/README.md`의 설치 가이드를 따랐는가?

**확인 방법:**
```bash
# Linux/Mac/Git Bash
ls -lh lib/xlsx.full.min.js

# Windows PowerShell
Get-Item lib\xlsx.full.min.js | Select-Object Name, Length
```

**예상 출력:**
```
-rw-r--r-- 1 user user 1.1M Oct 12 16:00 lib/xlsx.full.min.js
```

### 3. 브라우저 확인

- [ ] 최신 브라우저가 설치되어 있는가?
  - Chrome, Firefox, Edge, Safari 등
- [ ] JavaScript가 활성화되어 있는가?
- [ ] 로컬 파일 실행이 허용되어 있는가?

## 🧪 기능 테스트

### 테스트 1: 앱 실행

1. `index.html` 파일을 더블클릭하거나 브라우저에서 열기
2. 다음 항목 확인:
   - [ ] 페이지가 정상적으로 로드되는가?
   - [ ] 제목 "Excel 자동화 앱"이 표시되는가?
   - [ ] 파일 업로드 영역이 보이는가?
   - [ ] 사용 방법 안내가 표시되는가?

3. 브라우저 콘솔(F12) 확인:
   - [ ] 오류 메시지가 없는가?
   - [ ] "SheetJS 라이브러리가 로드되지 않았습니다" 오류가 없는가?
     - 있다면: 라이브러리를 다시 설치하세요

### 테스트 2: 샘플 Excel 파일 준비

1. Excel을 열고 새 통합 문서 생성
2. 시트 이름을 "Rawdata"로 변경
3. 다음 샘플 데이터 입력:

```
A1: 이름    B1: 나이    C1: 점수
A2: 홍길동  B2: 25      C2: 85
A3: 김영희  B3: 28      C3: 92
A4: 이철수  B4: 30      C4: 78
```

4. `test_sample.xlsx`로 저장
5. 확인:
   - [ ] 시트 이름이 정확히 "Rawdata"인가?
   - [ ] 첫 행이 헤더인가?
   - [ ] 데이터가 2행부터 시작하는가?

### 테스트 3: 파일 업로드

1. 웹 앱에서 파일 업로드 영역 클릭
2. `test_sample.xlsx` 파일 선택
3. 확인:
   - [ ] 파일이 정상적으로 업로드되는가?
   - [ ] "파일을 성공적으로 읽었습니다" 메시지가 표시되는가?
   - [ ] 오류 메시지가 없는가?

### 테스트 4: 데이터 통계

업로드 후 다음 항목 확인:

- [ ] "데이터 통계" 섹션이 표시되는가?
- [ ] 통계 카드들이 보이는가?
  - 총 데이터 행: 3
  - 총 컬럼 수: 3
  - 숫자 컬럼: 2 (나이, 점수)
  - 텍스트 컬럼: 1 (이름)

### 테스트 5: Rawdata 미리보기

- [ ] "Rawdata 미리보기" 섹션이 표시되는가?
- [ ] 테이블이 정상적으로 표시되는가?
- [ ] 헤더(이름, 나이, 점수)가 보이는가?
- [ ] 데이터 3행이 모두 표시되는가?

### 테스트 6: 생성된 보고서

- [ ] "생성된 보고서" 섹션이 표시되는가?
- [ ] 보고서 테이블이 보이는가?
- [ ] 각 컬럼별 통계가 표시되는가?
  - 총 데이터 수
  - 고유값 수
  - 빈값 수
- [ ] 숫자 컬럼 통계가 표시되는가?
  - 나이: 평균 27.67, 최소 25, 최대 30
  - 점수: 평균 85, 최소 78, 최대 92

### 테스트 7: 보고서 다운로드

1. "보고서 다운로드" 버튼 클릭
2. 확인:
   - [ ] 파일이 다운로드되는가?
   - [ ] 파일명이 `Report_[날짜시간].xlsx` 형식인가?
   - [ ] 다운로드된 파일을 Excel에서 열 수 있는가?
   - [ ] "Rawdata" 시트가 있는가?
   - [ ] "Report" 시트가 있는가?
   - [ ] 두 시트 모두 데이터가 올바른가?

## ⚠️ 문제 해결

### 문제 1: "SheetJS 라이브러리가 로드되지 않았습니다"

**증상:** 브라우저 콘솔에 오류 메시지

**해결:**
1. `lib/xlsx.full.min.js` 파일 크기 확인
2. 1KB 미만이면 stub 파일 → 실제 라이브러리로 교체
3. `lib/README.md`의 설치 가이드 참조
4. 라이브러리 다운로드 및 교체
5. 브라우저 새로고침 (Ctrl+F5)

### 문제 2: "Rawdata 시트를 찾을 수 없습니다"

**증상:** 파일 업로드 후 오류 메시지

**해결:**
1. Excel 파일 열기
2. 시트 탭 이름 확인
3. 정확히 "Rawdata"로 변경 (대소문자 정확히)
4. 파일 저장 후 다시 업로드

### 문제 3: 페이지가 로드되지 않음

**증상:** 빈 화면 또는 오류

**해결:**
1. 브라우저 콘솔(F12) 확인
2. JavaScript가 활성화되어 있는지 확인
3. 다른 브라우저에서 시도
4. 파일 경로에 한글이나 특수문자가 있는지 확인

### 문제 4: 파일 업로드가 안 됨

**증상:** 파일 선택 후 아무 반응 없음

**해결:**
1. 파일 형식 확인 (.xlsx 또는 .xls)
2. 파일이 손상되지 않았는지 확인 (Excel에서 열리는지)
3. 파일 크기 확인 (너무 크면 시간이 걸림)
4. 브라우저 콘솔에서 오류 확인

### 문제 5: 보고서 다운로드가 안 됨

**증상:** 다운로드 버튼 클릭 시 반응 없음

**해결:**
1. 브라우저의 팝업 차단 확인
2. 다운로드 폴더 권한 확인
3. 브라우저 콘솔에서 오류 확인
4. 다른 브라우저에서 시도
