# 빠른 시작 가이드 (Quick Start Guide)

Excel 자동화 앱을 5분 안에 시작하는 방법입니다.

## ⚡ 빠른 시작 (3단계)

### 1단계: SheetJS 라이브러리 설치

가장 중요한 단계입니다! 다음 중 **하나**의 방법을 선택하세요:

#### 방법 A: 직접 다운로드 (권장)

1. 브라우저에서 다음 링크 중 하나를 열기:
   - https://cdn.sheetjs.com/xlsx-0.20.0/package/dist/xlsx.full.min.js
   - https://unpkg.com/xlsx@0.18.5/dist/xlsx.full.min.js

2. 페이지의 모든 내용을 복사 (Ctrl+A, Ctrl+C)

3. 텍스트 편집기로 `lib/xlsx.full.min.js` 파일 열기

4. 기존 내용을 모두 삭제하고 복사한 내용 붙여넣기

5. 파일 저장

#### 방법 B: 명령줄 사용 (Linux/Mac/Windows Git Bash)

```bash
cd lib
curl -L -o xlsx.full.min.js https://cdn.sheetjs.com/xlsx-0.20.0/package/dist/xlsx.full.min.js
```

#### 방법 C: npm 사용

```bash
npm install xlsx
cp node_modules/xlsx/dist/xlsx.full.min.js lib/
```

#### ✅ 설치 확인

`lib/xlsx.full.min.js` 파일 크기가 **800KB ~ 1.2MB** 정도면 성공! 
(1KB 미만이면 stub 파일이므로 다시 설치 필요)

### 2단계: Excel 파일 준비

1. Excel 파일을 열거나 새로 만들기
2. 시트 이름을 **"Rawdata"**로 변경 (정확히 일치해야 함)
3. 첫 번째 행에 컬럼 헤더 입력
4. 두 번째 행부터 데이터 입력

**예시:**

```
A1: 이름       B1: 부서     C1: 나이    D1: 급여
A2: 홍길동     B2: 개발팀   C2: 30      D2: 5000000
A3: 김영희     B3: 영업팀   C3: 28      D3: 4500000
A4: 이철수     B4: 개발팀   C4: 35      D4: 6000000
```

5. `.xlsx` 형식으로 저장

### 3단계: 앱 실행

1. `index.html` 파일을 브라우저에서 열기
   - 파일 더블클릭
   - 또는 브라우저 > 파일 > 열기
   
2. 파일 업로드 영역 클릭 또는 드래그 앤 드롭

3. Excel 파일 선택

4. 자동으로 분석 및 보고서 생성 완료! ✨

5. "보고서 다운로드" 버튼으로 결과 저장

## 📊 무엇이 생성되나요?

앱은 자동으로 다음을 분석합니다:

### 데이터 통계
- 총 행 수
- 총 컬럼 수
- 숫자/텍스트 컬럼 구분

### 컬럼별 분석
- 총 데이터 수
- 고유값 개수
- 빈값 개수
- 숫자 컬럼: 최솟값, 최댓값, 평균, 합계
- 텍스트 컬럼: 상위 빈도값 Top 5

### 생성되는 보고서
- **Rawdata 시트**: 원본 데이터
- **Report 시트**: 생성된 분석 보고서

## 🎯 사용 예시

### 직원 급여 분석

**입력 (Rawdata 시트):**
- 직원 이름, 부서, 나이, 급여 데이터

**출력 (Report 시트):**
- 급여 평균: 5,166,667원
- 급여 범위: 4,000,000 ~ 6,000,000원
- 부서별 인원: 개발팀(3명), 영업팀(1명)
- 평균 나이: 30.75세

### 판매 실적 분석

**입력 (Rawdata 시트):**
- 날짜, 제품명, 카테고리, 수량, 단가, 총액

**출력 (Report 시트):**
- 총 판매액
- 제품별 판매 수량
- 카테고리별 통계
- 가장 많이 팔린 제품 Top 5

### 설문조사 결과 분석

**입력 (Rawdata 시트):**
- 응답자, 연령대, 성별, 만족도, 지역

**출력 (Report 시트):**
- 평균 만족도
- 연령대별 분포
- 지역별 응답 수
- 가장 높은 만족도 비율

## ⚠️ 자주 발생하는 오류

### 1. "Excel 라이브러리가 로드되지 않았습니다"

**원인:** SheetJS 라이브러리 미설치

**해결:**
- 1단계로 돌아가서 라이브러리 설치
- `lib/xlsx.full.min.js` 파일 크기 확인 (800KB 이상)
- 브라우저 새로고침

### 2. "Rawdata 시트를 찾을 수 없습니다"

**원인:** 시트 이름이 "Rawdata"가 아님

**해결:**
- Excel에서 시트 탭 이름 확인
- 정확히 "Rawdata"로 변경 (대소문자 정확히)
- "Raw data", "rawdata", "RAWDATA" 모두 불가

### 3. "데이터가 없습니다"

**원인:** Rawdata 시트가 비어있음

**해결:**
- 최소 2행 이상 필요 (헤더 + 데이터 1행 이상)
- 첫 행이 헤더인지 확인
- 빈 행으로 시작하지 않는지 확인

### 4. 파일 업로드가 안 됨

**원인:** 파일 형식 문제

**해결:**
- `.xlsx` 또는 `.xls` 형식인지 확인
- 다른 이름으로 저장 시도
- Excel 파일이 손상되지 않았는지 확인

## 💡 유용한 팁

### 빠른 테스트
- 5-10행 정도의 작은 데이터로 먼저 테스트
- 정상 작동 확인 후 실제 데이터 사용

### 데이터 준비
- 컬럼 이름은 명확하게 (한글 가능)
- 각 컬럼은 하나의 데이터 타입만 사용
- 빈 셀 최소화

### 성능 향상
- 불필요한 컬럼 제거
- 10만 행 이하 권장
- 대용량 데이터는 분할 처리

### 보안
- 데이터는 브라우저에서만 처리됨
- 서버 전송 없음
- 내부망에서 안전하게 사용 가능
