# Excel 자동화 앱 (Excel Automation App)

엑셀 파일의 Rawdata 시트에서 데이터를 읽어 자동으로 보고서를 생성하는 웹 기반 애플리케이션입니다.

## 주요 기능

- 📊 Excel 파일 업로드 및 자동 분석
- 📈 Rawdata 시트의 데이터를 자동으로 읽고 처리
- 📋 자동 보고서 생성 (통계, 빈도 분석 등)
- 💾 생성된 보고서를 Excel 파일로 다운로드
- 🔒 내부망 환경에서 오프라인 작동 (Python 불필요)
- 🎨 직관적인 사용자 인터페이스

## 시스템 요구사항

- 최신 웹 브라우저 (Chrome, Firefox, Edge 등)
- JavaScript 활성화
- **중요:** SheetJS 라이브러리 필요 (lib/README.md 참조)

## 설치 방법

### 1. 저장소 다운로드

```bash
git clone https://github.com/yji0728/excelauto.git
cd excelauto
```

### 2. SheetJS 라이브러리 설치

**중요:** 애플리케이션을 사용하기 전에 반드시 SheetJS 라이브러리를 설치해야 합니다.

#### 방법 A: 인터넷 연결 가능한 환경

1. 다음 중 하나의 링크에서 라이브러리 다운로드:
   - https://cdn.sheetjs.com/xlsx-0.20.0/package/dist/xlsx.full.min.js
   - https://unpkg.com/xlsx@0.18.5/dist/xlsx.full.min.js

2. 다운로드한 파일을 `lib/xlsx.full.min.js`로 저장 (기존 파일 덮어쓰기)

#### 방법 B: 명령줄 사용 (Linux/Mac/Git Bash)

```bash
cd lib
curl -L -o xlsx.full.min.js https://cdn.sheetjs.com/xlsx-0.20.0/package/dist/xlsx.full.min.js
```

#### 방법 C: npm 사용

```bash
npm install xlsx
cp node_modules/xlsx/dist/xlsx.full.min.js lib/
```

#### 방법 D: 내부망 환경

1. 외부 인터넷이 가능한 PC에서 위 방법으로 다운로드
2. USB 또는 승인된 파일 전송 방법으로 내부망으로 전송
3. `lib/` 폴더에 `xlsx.full.min.js` 파일 배치

자세한 내용은 `lib/README.md` 파일을 참조하세요.

### 3. 애플리케이션 실행

브라우저에서 `index.html` 파일을 열면 됩니다:

- **방법 1:** 파일 탐색기에서 `index.html` 파일을 더블클릭
- **방법 2:** 브라우저에서 `File > Open` 메뉴로 파일 열기
- **방법 3:** 로컬 웹 서버 사용 (선택사항)

```bash
# Python 3가 설치된 경우 (선택사항)
python -m http.server 8000

# Node.js가 설치된 경우 (선택사항)
npx http-server

# 그 후 브라우저에서 http://localhost:8000 접속
```

**참고:** 로컬 파일로도 완벽하게 작동하므로 웹 서버는 필수가 아닙니다.

## 사용 방법

### 1. Excel 파일 준비

Excel 파일에 **"Rawdata"** 라는 이름의 시트를 생성하고 데이터를 입력합니다:

```
예시 (Rawdata 시트):
| 이름    | 부서   | 나이 | 급여     | 입사일      |
|---------|--------|------|----------|-------------|
| 홍길동  | 개발팀 | 30   | 5000000  | 2020-01-15  |
| 김영희  | 영업팀 | 28   | 4500000  | 2021-03-20  |
| 이철수  | 개발팀 | 35   | 6000000  | 2019-06-10  |
```

**중요사항:**
- 시트 이름은 반드시 "Rawdata"여야 합니다 (대소문자 구분)
- 첫 번째 행은 컬럼 헤더(제목)여야 합니다
- 데이터는 두 번째 행부터 입력합니다

### 2. 파일 업로드

1. 웹 애플리케이션을 실행 (`index.html` 열기)
2. 파일 업로드 영역을 클릭하거나 Excel 파일을 드래그 앤 드롭
3. `.xlsx` 또는 `.xls` 파일 선택

### 3. 자동 보고서 생성

파일이 업로드되면 자동으로:
- 데이터 통계 표시 (총 행, 컬럼 수 등)
- Rawdata 미리보기 (처음 100행)
- 자동 보고서 생성:
  - 각 컬럼별 통계 (데이터 수, 고유값, 빈값)
  - 숫자 컬럼: 최솟값, 최댓값, 평균, 합계
  - 텍스트 컬럼: 빈도 분석 (상위 5개)

### 4. 보고서 다운로드

- "보고서 다운로드" 버튼을 클릭
- `Report_[타임스탬프].xlsx` 파일이 다운로드됩니다
- 다운로드된 파일에는 2개의 시트가 포함됩니다:
  - **Rawdata**: 원본 데이터
  - **Report**: 생성된 보고서

## 특징

### ✅ 장점

- 설치 불필요 (브라우저만 있으면 됨)
- Python, Java 등 별도 런타임 불필요
- 내부망 환경에서 완전 오프라인 작동
- 데이터가 서버로 전송되지 않음 (완전한 로컬 처리)
- 크로스 플랫폼 (Windows, Mac, Linux)
- 직관적이고 깔끔한 UI

### 🔒 보안

- 모든 데이터 처리는 브라우저 내에서만 이루어짐
- 서버로 데이터 전송 없음
- 내부망 환경에 최적화

### ⚡ 성능

- 최대 수십만 행의 데이터 처리 가능
- 브라우저 메모리에 따라 성능 차이 발생 가능
- 대용량 파일의 경우 처리 시간이 소요될 수 있음

## 제한사항

1. Excel 파일에 반드시 "Rawdata" 시트가 있어야 합니다
2. 브라우저 메모리 제한에 따라 처리 가능한 파일 크기가 제한될 수 있습니다
3. 매크로나 수식은 처리되지 않으며, 값만 읽어옵니다
4. SheetJS 라이브러리가 없으면 작동하지 않습니다

## 문제 해결

### "Excel 라이브러리가 로드되지 않았습니다" 오류

**원인:** SheetJS 라이브러리가 설치되지 않았거나 경로가 잘못됨

**해결 방법:**
1. `lib/README.md` 파일의 설치 가이드를 따라 라이브러리 다운로드
2. `lib/xlsx.full.min.js` 파일이 존재하는지 확인
3. 파일 크기가 약 800KB ~ 1.2MB인지 확인 (너무 작으면 stub 파일)
4. 브라우저 콘솔(F12)에서 오류 메시지 확인

### "Rawdata 시트를 찾을 수 없습니다" 오류

**원인:** Excel 파일에 "Rawdata" 시트가 없음

**해결 방법:**
1. Excel 파일을 열고 시트 이름 확인
2. 시트 이름을 "Rawdata"로 변경 (대소문자 정확히 일치)
3. 데이터가 있는 시트의 이름을 "Rawdata"로 변경

### 파일이 업로드되지 않음

**원인:** 파일 형식이 지원되지 않거나 손상됨

**해결 방법:**
1. `.xlsx` 또는 `.xls` 형식인지 확인
2. Excel에서 파일을 다시 저장
3. 다른 브라우저에서 시도
4. 브라우저 콘솔(F12)에서 오류 메시지 확인

### 대용량 파일 처리 시 느림

**원인:** 브라우저의 메모리 제한

**해결 방법:**
1. 데이터를 분할하여 처리
2. 불필요한 컬럼 제거
3. 브라우저 메모리 확보 (다른 탭 닫기)
4. 더 많은 RAM을 가진 PC 사용
